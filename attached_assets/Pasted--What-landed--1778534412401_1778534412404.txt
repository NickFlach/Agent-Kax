  The verification math for the agent flow, end to end:

  1. User claims bot X. Server issues KAX-VERIFY-A3F7B2, stores {challenge, claim_subject=X, expires_at=now+30m}.
  2. User publishes an OBC artifact whose description includes KAX-VERIFY-A3F7B2.
  3. User POSTs {obcBotId: X, artifactUuid: Y} to verify.
  4. Server fetches artifact Y via partner API → checks creator_bot_id == X (proves ownership), description contains KAX-VERIFY-A3F7B2 (proves the user knows the challenge), artifact.created_at >=
  challenge.created_at (prevents replay of pre-existing artifacts), challenge unconsumed, challenge unexpired.
  5. All five checks pass → consume challenge atomically, upsert user, issue session.

  Deploy steps for you when pulling:
  1. pnpm install (adds ethers@^6)
  2. Apply migration: psql $DATABASE_URL -f lib/db/migrations/0001_wallet_and_agent_auth.sql
  3. Set KAX_CORS_ALLOWLIST env (the CORS fix from the earlier commit) to include your wallet UI origin
  4. Restart api-server

  The flows are testable via curl / Postman today — no UI required for verification. Once you confirm wallet + agent both produce session cookies, I'll write the SPA login page in the next pass.
